# Lessons Learned

这里存放不适合继续堆进 `CLAUDE.md` 的内容：非显而易见的实现决策、跨模块契约、重要边界条件和长期维护经验。日常协作先看 `CLAUDE.md`，遇到细节问题再查这里；ADR 历史与提案统一放在 `docs/ADR/`；当前部署状态和数据规模看 `docs/PROJECT_STATUS.md`。

## 使用约定

- 只记录"以后还会反复用到"的经验；一次性背景、显而易见的代码事实、API 枚举不要堆进来。
- **按主题合并，而不是按时间追加**。新条目优先进已有主题，没有合适主题时才新增。
- 如果某条已经进入 `CLAUDE.md` / ADR，或代码本身已经足够自解释，就从这里删掉。
- 不写"现在跑几个实例"、"导了多少数据"、"什么时候全量更新"这类快照类信息——那是 `docs/PROJECT_STATUS.md` 的事。
- 不写"我们打算怎么搞"的提案——那是 ADR 的事。
- **单文件目标 ≤ 600 行**；超过 350 行先重组章节，超过 600 行按主题拆独立文件（如 `lesson_learned_search.md`）。

## ADR 导航

## 维护触发器

- 写完一条：找找能不能并入已有条目
- 单主题行数 > 80：拆子章节
- 全文件 > 350 行：重组章节
- 全文件 > 600 行：按主题拆独立文件
- 某条内容 1 年内没再被引用：考虑删除或迁入 ADR
